==== Migrating from chart version <= 0.1.5 ====
Prior chart versions shipped CRDs via the Helm `crds/` directory, which never
upgraded them on `helm upgrade`. Starting with 0.2.0 CRDs are templates and
Helm manages their lifecycle.

If `helm upgrade` fails with "invalid ownership metadata" on a CRD, take
ownership of the existing CRDs before re-running:

  for crd in $(kubectl get crd -o name | grep kubeswarm.io); do
    kubectl annotate "$crd" meta.helm.sh/release-name={{ .Release.Name }} --overwrite
    kubectl annotate "$crd" meta.helm.sh/release-namespace={{ .Release.Namespace }} --overwrite
    kubectl label    "$crd" app.kubernetes.io/managed-by=Helm --overwrite
  done

Alternative escape hatch: `helm upgrade ... --set crds.install=false` and
apply CRDs manually from the operator repo.
